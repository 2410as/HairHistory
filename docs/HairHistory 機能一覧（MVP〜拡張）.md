# HairHistory 機能一覧（MVP〜拡張）

HairHistory で実装したい機能と、着手順の整理メモです。

- --

## 1. コア機能（MVP）

### 1.1 髪の履歴管理

- **履歴一覧表示**

- ユーザーごとの履歴一覧（新しい順）

- 行ごとの表示項目: 日付 / 施術内容（カラー・ブリーチ・縮毛矯正など）/ サロン名 / スタイリスト名 / ダメージメモの要約

- **履歴登録**

- 入力項目:

- 日付

- 施術内容（カラー / ブリーチ / 縮毛矯正 / トリートメント / その他）

- サロン名（フリーテキスト）

- スタイリスト名（フリーテキスト）

- メモ（ダメージ・薬剤・注意点など）

- **履歴編集**

- 既存の履歴の内容を修正できる

- **履歴削除（任意）**

- 間違えて登録した履歴を削除できる

### 1.2 匿名ユーザー & 識別

- **匿名ユーザーID発行**

- 初回アクセス時にランダムな `userId` を発行

- ブラウザ（localStorage など）に保存

- **ユーザーごとの履歴ページ**

- `userId` に紐づいた履歴一覧ページ

### 1.3 QRコード共有（URL共有のUI表現）

- **QRコード生成**

- 自分の履歴ページURLを QR コードとして画面に表示

- **美容師との共有フロー**

- ユーザーが QR コードを見せる

- 美容師がスマホで読み取って、同じ履歴一覧ページを開く

- --

## 2. ダメージ把握を助けるUI

- **ダメージメモの書きやすさ**

- メモ欄プレースホルダー例:

- 「毛先がかなりダメージ / ブリーチ〇回目 / 薬剤がしみた など」

- **直近履歴の強調表示**

- 直近3〜5件の履歴をカード風に大きく表示

- **ざっくりダメージサマリー（将来的な拡張）**

- 一覧の1行に「ブリーチ通算○回目」「前回縮毛矯正から○ヶ月」などを表示

- --

## 3. 非機能・運用まわり

- **ログインなしで利用可能（ゲスト利用）**
- **スマホ最適化されたUI（レスポンシブ対応）**
- **データ永続化**

- DB（PostgreSQL）に `users` / `hair_histories` テーブル

- **エラーハンドリング**

- APIエラー時のメッセージ表示（最低限）

- --

## 4. 着手順メモ（バックエンド → フロント）

1. **バックエンド**

- Go API プロジェクト作成

- `users` / `hair_histories` のテーブル定義（SQL）

- API:

- `POST /api/users`（匿名ユーザーID発行）

- `GET /api/users/{userId}/histories`（一覧）

- `POST /api/users/{userId}/histories`（作成）

- `PUT /api/histories/{historyId}` / `DELETE /api/histories/{historyId}`（必要なら）

2. **フロント（Next.js）**

- 初回アクセス時の `userId` 生成＆保存

- 履歴一覧ページ + 履歴登録フォーム

3. **QRコード共有**

- 履歴一覧ページのURLから QR コード生成・表示

4. **ダメージUI強化**

- 直近履歴カード、ダメージサマリー表示など
