# HairHistory 要件定義

このファイルは「何を作るか」を決めるための最小ドキュメントです。  
実装の細かい設計は `HairHistory 基本設計（ドラフト）.md` を参照してください。

---

## 1. 目的

- 髪の履歴（カラー・ブリーチ・縮毛矯正など）を記録する
- 美容師さんが履歴を確認しやすくする
- 次回施術の判断をしやすくする

## 2. 対象ユーザー

- 記録する人: 施術履歴を残したい本人
- 確認する人: 施術前に履歴を確認したい美容師さん

## 3. MVPでやること

- 匿名ユーザーIDを発行する（ログインなし）
- 履歴を追加・一覧・編集・削除できる
- 共有用URL / QRで履歴を見せられる

### 履歴1件の項目（MVP）

- 日付（必須）
- 施術内容（必須）
- サロン名（任意）
- スタイリスト名（任意）
- メモ（任意）
- 画像は扱わない

## 4. 非機能（MVP）

- Webブラウザで利用（PC / スマホ）
- HTTPS前提
- DBで永続化（PostgreSQL）

## 5. 現時点の決定

- 認証は後回し（まずはログインなし）
- サロン / スタイリストは毎回フリーテキスト入力
- 詳細な検索・分析は拡張で対応

## 6. 技術方針（現時点）

- フロント: Next.js + React
- API: Go
- DB: PostgreSQL
- デプロイ: Railway（MVP）
