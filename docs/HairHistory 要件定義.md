# HairHistory 要件定義

## 1. プロジェクト概要

- **プロジェクト名**: HairHistory
- **目的**: 髪の履歴（ヘアカラー・ブリーチ・縮毛矯正など）を「メモ」として残し、**髪へのダメージ状況を美容師が把握しやすくする**。ユーザー自身も履歴を振り返り、次回以降の施術方針を一緒に考えやすくする。
- **想定プラットフォーム**: **Webアプリ**

## 2. ステークホルダー・ユーザー像

- **プロダクトオーナー**: 

- **記録する人**: 自分で「いつ・何をしたか」をメモし、次回の施術やセルフケアに活かしたい人

- **確認する人**: 美容師さんが、お客様の髪の履歴（カラー・ブリーチ・縮毛矯正など）を把握し、安全かつ適切に施術するために参照する

→ 自分で記録しつつ、美容師さんが髪を扱うときに確認しやすいUI・情報設計にする。

## 3. 最初に実現したいこと（MVP）

- **いつ**施術したか（日付）を記録する
- **どんな施術**をしたか記録する

- ヘアカラー

- ブリーチ

- 縮毛矯正

- パーマ

- （その他メモ可能）

- サロン**と**スタイリスト**を紐づけて保存する

## 4. ユースケース（ざっくり）

- **UC-1**: 施術履歴を登録する（日付・施術内容・サロン・スタイリスト・メモ）
- **UC-2**: 過去の施術内容（ヘアカラー / ブリーチ / 縮毛矯正など）を一覧・詳細で確認する
- **UC-3**: サロン・スタイリスト情報を登録し、履歴に紐づける
- **UC-4**: 美容師さんが、お客様の髪の履歴を確認しやすい形で閲覧する（今後の検討）
- **UC-5**: （追記予定）

## 5. 機能要件（ドラフト）

※MVPを中心に具体化

### 5.1 アカウント・認証

- **方針（MVP）**: **最初はログインなし**。いずれ追加する
- **美容師への共有方法**:

- **QRコード**を1つ発行し、そのQRコードを美容師と共有する。自分の履歴をまとめて見せる

- 共有用QRコードでそのページを見せられるようにする（QRコードを共有すれば美容師がブラウザで閲覧可能）

- スマホを渡して見せる（同じ画面をそのまま見せられる想定）

- 将来的にログイン を導入するかは後で検討

### 5.2 髪の履歴管理（MVP コア）

- **1件の履歴に含めたい項目**:

- **日付**（いつ施術したか）※必須想定

- **施術内容**: ヘアカラー / ブリーチ / 縮毛矯正 など（複数選択 or メモ）

- **サロン名**（フリーテキスト）

- **スタイリスト名**（フリーテキスト、または担当者）

- **メモ**（色番号・薬剤名・気になったことなど）

- 写真は扱わない**（MVP では画像保存なし）

- サロン・スタイリスト**: 毎回フリーテキストで入力。マスタ登録・選択は行わない（MVP）

### 5.3 検索・閲覧

- タイムライン or カレンダー表示（新しい順で見返せる）
- 条件検索（施術種別、期間、サロン など）は MVP でどこまで必要か要検討

### 5.4 通知・リマインド（必要なら）

- カット・カラー時期のリマインドなど（後回しでも可）

### 5.5 その他

- 多言語対応の有無
- ダークモード対応の有無
- **美容師が確認しやすいUI**: 一覧で「直近のカラー・ブリーチ・縮毛矯正」がパッと分かるようにするかなど

## 6. 非機能要件（ドラフト）

- **対応**: Webアプリ → PC / スマホ / タブレットのブラウザで利用
- **パフォーマンス**: （例）メイン画面は3秒以内に表示 など
- **セキュリティ**: パスワード保存方法、通信のHTTPS化 など
- **バックアップ・復元**: データ消失時の対応
- **インフラ・コスト**: 個人開発のため、**Railway など有料プラン前提のサービスは避け、無料枠で運用できるサービス（例: Vercel 無料枠 / Render 無料枠 / Supabase 無料枠 など）を優先して採用する。**
